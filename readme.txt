1. 前后台完全分离，支持前后台不同域名

2. 需要表字段

用户表：user_id,openid,addr,user_name,phone,money,share（分享次数）,flag(微信用户为1 ,短信用户为2)

money>0 玩过一次，
money =0 有机会玩，没有点中就再来一次


金额累加，每次使用 大于30扣 30 ,小于30扣 实际

分享一次，share+1 ，使用了 扣除一次

订单表：

order_id,goods_id,status(发货状态),order_sn(订单号),delivery_sn(快递单号),user_id(下单人),addr,phone,total_amount(总的需要支付金额),real_amount(实际需要支付金额),discounted_amount(抵扣金额)


商品表：
goods_id, goods_pic，goods_price,goods_name

3.接口

3.1 【开始页】判定用户是否玩过游戏，是否分享过

是否有订单，有订单先显示 订单发货状态

再判定如下：

玩过游戏，没有分享过: 显示已有金额 + 分享按钮 + 购买按钮
玩过游戏，分享过：    显示已有金额 + 再玩一次+ 分享按钮 + 购买按钮

没有玩过,没有分享： 游戏界面 
没有玩过，分享过： 显示 可玩次数 + 分享按钮 + 购买按钮


3.2 【中奖页】 存储中奖信息 

微信用户：存储中奖信息 ,用户表中：openid,money
短信用户：不调用

页面保存

3.3 【地址页】 存储收货信息+ 金额
 微信用户： 存订单表，存user表
 短信用户： 同上，只是新建用户数据，以phone为唯一项，用本地mac为标识



4. 开始页- 游戏中-中奖页-商品页-地址页-开始页

   开始页- 游戏中-中奖页-分享页-地址页-开始页
